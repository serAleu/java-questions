*Анонимные вложенные классы*

1) Чтобы понять, как работают анонимные классы я разбил объяснение на 2 шага, в первом шаге
(package step1) указано как работает без анонимных классов, на втором шаге (package step2)
мы рассмотрим как то же самое работает с использованием анонимных классов.

*step1:
    Предположим у нас задача сделать 3 модуля мониторинга которые будут отслеживать какие-то
    параметры с момента запуска программы. Без использования анонимных классов мне нужно
    создать 1 интерфейс, 3 класса, а потом, при старте программы вызвать методы мониторинга.
    Получается как то дофига ради единственного вызова 3ех методов.

*step2:
    В этом шаге мы рассматриваем как оптимизировать код представленный в step1 с помощью
    анонимных классов. Иди в код там понятнее.

2) Анонимный класс - это полноценный внутренний класс, поэтому, у него есть доступ к
переменным внешнего класса, в том числе к статическим и приватным.

3) Анонимные классы видны только внутри того метода, в котором они определены.

4) Анонимные классы не могут содержать статический контекст.

5) Помни, что для написания анонимного класса, наличие необходимого интерфейса - это ключевой
момент.
