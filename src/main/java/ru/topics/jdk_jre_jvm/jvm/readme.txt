*JVM*

1) JVM имеет две основные функции:
    - Позволяет запускать Java-приложения на любых устройствах или операционных
системах (принцип — «Написал один раз, запускай везде»).
    - Управляет, оптимизирует и очищает память, используемую приложением.
Garbage Collector - это часть JVM которая отвечает за сборку мусора. (Проверяет
есть ли в стеке ссылки на объект если нет, удаляет его из кучи).

2) Java - это компилируемый язык, это значит, что код, написанный на java,
при запуске программы переводится в байт-код с помощью JVM. Только после этого
байт-код переводится в машинный код для конкретного процессора.

          байт-код          машинный код
JAVA CODE ============> JVM ============> ПРОЦЕССОР

2.1) Именно из-за подхода описанного в пункте 2, программы на JAVA могут быть
запущены на разных устройствах.

3) JVM содержит интерпретатор байт-кода который переводит байт-код в машинный код.

4) JVM содержит JIT (Just-in-time compiler) который оптимизирует работу
интерпретатора байт-кода.

5) В open-source продуктах часто используется реализация виртуальной машины
JVM HotSpot.
