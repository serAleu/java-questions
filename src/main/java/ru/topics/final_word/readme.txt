*Наследование в java*

1) final класс НЕ может иметь наследников, это полезно при создании неизменяемых объектов
т.е. immutable.

2) final метод не может быть переопределен в подклассах, это полезно когда мы хотим, что бы
исходную реализацию метода нельзя было переопределить.

3) final переменную можно инициализировать только 1 раз, сразу же после создания или потом.
Уточнение - если тип данных на которые ссылается final переменная неизменяемый (immutable),
то изменить значение такой переменной нельзя (String, Integer и тп.). Если тип изменяемый
(mutable), то изменить значение такой переменной можно в куче, а ссылку все равно нельзя.
Например, класс StringBuilder mutable тип, изменить значение его переменной после
инициализации можно с помощью метода append(), см. пример.

4) effectively final - это переменная значение которой не меняется после инициализации,
даже если она не объявлена как final.
